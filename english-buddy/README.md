# English Buddy 🗣️ — 給 9 歲孩子的情境式英語口說陪練

一套以**口說對話**為主、**情境式**、可**切換場景**的英語學習雛形。
語音為主、家長陪同使用。這是 v0.1 prototype，目的是**先驗證孩子愛不愛玩、哪種場景有效**，再決定要不要長出完整 App。

> 這個資料夾與 `cebu-trip` 完全獨立，互不干擾。

---

## 一、設計第一原理：把「反覆使用 → 內化」做成系統

這套系統的核心來自一個觀察：

> 當你反覆地、有目的地在真實任務裡使用某些字（skill / timeline / goal），
> 你對它們的敏感度會變高，最後內化成自己的能力——這是「習得」，不是「背誦」。

對 9 歲孩子，我們把它拆成三個機制，直接寫進系統：

| 機制 | 做法 |
|---|---|
| **高頻曝光** | AI 在對話中自然、反覆地說出目標語塊 |
| **有目的地使用** | 每個場景是一個「要用英語才能完成的小任務」，不是背單字 |
| **跨情境重複（螺旋複習）** | 系統把**舊場景**學過的語塊，刻意塞進**新場景**，製造反覆內化 |

並且：
- 教的是**整句語塊**（"Can I have ...?"），不是單字——更接近真實口說，也更黏。
- 每個語塊都記錄「**曝光次數**」與「**孩子主動說出次數**」，越用越熟、進度看得見。

---

## 二、一次對話的循環（10–15 分鐘）

1. **暖身**：AI 用孩子學過的語塊打招呼。
2. **任務**：給一個小目標（例：跟店員點一球巧克力冰淇淋並問價錢）。
3. **鷹架**：孩子卡住 / 講中文時，AI 給一個簡單選項或**一句中文小提示**，熟了就撤掉。
4. **偷渡複習**：AI 在新場景自然重複高價值的舊語塊。
5. **收尾**：標記今天「真的說出口」的語塊 → 看得見的成就感。

## 三、怎麼用（家長）

語音（麥克風）需要**安全環境**才能啟用，所以不要直接用 `file://` 打開，請用本機伺服器：

```bash
cd english-buddy
python3 -m http.server 8000
# 然後用 Chrome 開 http://localhost:8000
```

1. 第一次進入 → 點右上「⚙️ 設定」，貼上你的 **Anthropic API key**（只存在這台電腦的瀏覽器 localStorage，不會上傳）。
   - 申請：<https://console.anthropic.com>
2. 選一個場景 → AI 會先開口（並用語音念出來）。
3. 孩子按 **🎤 我要說** 對著麥克風用英語回答；說不出來時可改用打字框。
4. 卡住時你在旁邊引導：可以指右側的「目標語塊」提示他可以說哪一句。
5. 右側進度條會記錄每個語塊「聽過幾次 / 說出幾次」。

> **建議**：每天 10–15 分鐘 > 一週一次一小時。先玩 2–3 個場景，觀察他對哪種最有反應。

## 四、檔案結構

```
english-buddy/
├── README.md      ← 你正在看的概念 + 使用文件
├── index.html     ← 介面
├── styles.css     ← 樣式
├── scenes.js      ← 場景與目標語塊資料（最容易擴充的地方）
└── app.js         ← 對話邏輯、語音、進度、螺旋複習
```

**想新增場景？** 只要在 `scenes.js` 的陣列裡多加一個物件即可，不用動其他檔案。

## 五、Roadmap（先驗證，再長大）

- **v0.1（現在）**：單頁語音網頁、6 個場景、語塊進度、螺旋複習。家長陪同。← 你在這
- **v0.2**：每日連續紀錄、語塊「熟練度」分級（聽過 → 說過 → 熟練）、更多場景、孩子自選頭像。
- **v0.3**：家長後台（看哪些語塊卡住）、把 API key 移到後端代理（不放瀏覽器）、帳號與長期進度。
- **未來**：發音回饋、語塊間自動排程複習（間隔重複）、孩子自己用的「免家長」模式。

## 六、已知限制（雛形階段）

- API key 放在瀏覽器只適合**自家、單機**使用；要上線給別人用，務必改成後端代理。
- 瀏覽器語音辨識（Web Speech API）在 **Chrome** 上最穩；Safari/Firefox 支援度不一，必要時用打字框。
- 還沒有發音正確度評分——目前重點是「敢開口、反覆說」。
